# ディレクトリ構造
- vite.config.js: Viteの設定ファイル。新規でプラグインを追加する等しない限り特に触る必要はない
- package.json: どのライブラリをどのバージョンで使うか、どんなコマンドが使えるかが書いてある
- package-lock.json: 依存関係をより厳密に管理する為のファイルで自動生成されるが、これも普通は直接触らない
- index.html: 触らなくてもいいがタイトルを変えたりは可能。<div id="root">のようにReactのJSコードが生成され、差し込む場所になる要素が記述されている
- eslint.config.js: コードの書き方のルールやフォーマットを指定する設定ファイル
- srcフォルダ: 基本Reactのコードを書くフォルダ
  - main.jsx: Reactのエントリーポイントで、このファイルがindex.html内の<div id="root">に対してコンポーネントをレンダリングする
  - App.jsx: トップレベルのコンポーネント
  - assetsフォルダ: jsで読み込む外部ファイル(css等)の置き場
- publicフォルダ: 静的ファイル(画像や音楽、フォント等)を置く場所

## 環境構築(オプション)
ケアレスミスを減らすために、「eslint.config.js」の設定を入れておく  
ESlintはバリデーションツールで、コードミスや非推奨な書き方をすると自動で教えてくれるツール  
- no-undef: 未定義の変数を使用している場合にエラーを出して知らせるように設定
- no-unused-vars: 定義したけど使用していない変数がある場合にエラーを出して知らせるように設定
- react/prop-types: React特有の型定義ルールがあるが、現在はこれを使うくらいならTypeScriptを使って実装するが、今回は使わないのでオフ
```
rules: {
      ...js.configs.recommended.rules,
      ...react.configs.recommended.rules,
      ...react.configs['jsx-runtime'].rules,
      ...reactHooks.configs.recommended.rules,
      'react/jsx-no-target-blank': 'off',
      'react-refresh/only-export-components': [
        'warn',
        { allowConstantExport: true },
      ],
+     "no-undef": "error",
+     "no-unused-vars": "error",
+     "react/prop-types": "off",
    },
```
